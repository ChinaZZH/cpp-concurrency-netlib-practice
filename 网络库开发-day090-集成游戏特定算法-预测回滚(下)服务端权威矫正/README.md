# Day90： 集成游戏特定算法  预测回滚(下) -服务端权威矫正

## 核心收获

-- 1.  客户端预测的时候会先进行模拟，然后才发ClientInput包，这个数据包带着预测后的坐标x, y。
 
-- 2.  服务端接受到预测包存起来。然后并且也每20毫秒执行一次 每个在线玩家也执行一次模拟预测(和客户端同步) ，然后如果坐标的差值超过阈值则下发 更正包给客户端。
 
-- 3.  客户端收到更正包，进行服务端权威覆盖。并且把帧号大于当前服务端确认的帧号的操作记录进行回放形成最新的记录。


## 数据库时间点恢复的比较
|数据库原理 |	客户端预测实现+回滚重放|	作用|
|------------|-----------------------------|-------|
|全量备份（Full Backup）|	snapshots_.Save() 保存的 PlayerSnapshot	| 某一帧时刻的完整世界状态（位置、速度、血量）|
|增量日志（Binlog / Redo Log）|	pending_inputs_（std::map 中的输入帧）	| 按时间顺序（帧号）排列的所有操作记录|
|恢复工具（mysqlbinlog）|	Rollback(target_frame) + Replay()	 | 从全量备份出发，按顺序重放日志，将状态“快进”到最新时刻|
|日志清理（Purge）|	OnServerFrame 中 erase(<= acked_frame) |	确认该帧已被服务端归档（落盘/广播），本地可以安全删除|
|主从同步校验	| “服务端权威矫正”	| 主库（服务端）算出权威结果，对比从库（客户端）的查询结果是否一致|
