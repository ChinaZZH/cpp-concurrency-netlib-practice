## monkey编译器

## 核心收获

-- 1.1. 定义第一条字节码指令并实现Make指令编码器

-- 2.1. 编码器支持中缀表达式

-- 2.2. 通过Dissamble支持反汇编，将字节码翻译成中间语言(汇编语言)

-- 2.3. 支持编译，将语法树的结点node根据不同的语句和表达式转化为汇编语言并且同时格式化为字节码

-- 2.4. 编译完成后，虚拟机运行的时候要从这个编译对应中获取编译完成的字节指令码 和 常量池。

-- 3.1. 支持前缀表达式的的编译和字节码生成

-- 3.2. 支持字面量表达式的的编译和字节码生成

-- 4.1. 支持编译if语句

-- 4.2. 编译if语句，需要支持 条件为假的时候跳转(code.OpJumpNotTruthy) 和 无条件跳转(code.OpJump)

-- 4.3. 由于写入跳转语句的时候还不知道具体要跳转到哪里，所以需要先记录当前序列的位置，当到底需要跳转的地址的时候再把该地址回填。

-- 4.4. 跳转采用的是相对偏移位置的跳转。就是 跳转语句结束后还需要跳过多少字节的指令才到 需要执行的执行的位置。(offset)

-- 5.1. 生成指令设置全局变量和获取全局变量

-- 5.2. 引入符号表(SymbolTable) 并且支持 let 语句 和标识符表达式。

-- 5.3. 搭建虚拟机基础框架搭建 并且让虚拟机支持之前所有的指令码同时虚拟机支持全局变量。

-- 5.101.  定义frame支持每个函数调用环境，并且vm会定义 frame数组来定义  函数调用栈帧，标注当前所执行的函数frameIdex.

-- 5.102.  编译函数字面量的时候需要支持隐式返回，通过return指令来处理函数帧栈的出栈操作。

-- 5.103.  编译return指令面对return ; 的时候需要插入一条code.OpNull， 这个指令就是往操作数栈里面插入null 对象。

				monkey设计任何表达式都需要返回值， 支持用if语句，let语句各种语句使用嵌入函数调用。  插入 OpNull 是为了保证虚拟机执行模型的一致性。
				
-- 5.104.  编译函数字面量的时候就相当于生成一个常量，只不过这个常量需要额外处理一些其他内容，所以用code.OpClosure指令和code.OpConstant和常量处理区分开来。


-- 6.1.  将具体的数组，哈希表，字符串 生成对应的字面量常量存入常量表中 并生成对应的指令。

-- 6.2.  通过对应的指令从vm操作数栈中取出数组，哈希表，字符串对应的数值。

-- 6.3.  索引操作根据对应的 操作数 和操作符索引 取出对应的内容。


-- 7.1.  当标识符在符号表中找不到对应的符号对象的时候，这个时候就查下内置函数表。

-- 7.2.  内置函数表找到了对应的内置函数对象，则将内置放入常量表中。并且在指令集中放入opConstant指令。

-- 7.3.  运行常量指令的时候会先将函数常量放入操作数栈

-- 7.4.  在运行函数调用指令的时候判断函数名是否是内置函数对象，是的话通过参数执行直接出结果。

-- 9.1. 使用对象池内存池来控制函数帧的分配和回收，避免频繁创建或者销毁内存。
