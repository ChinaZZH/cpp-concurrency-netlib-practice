# Day82： 集成游戏特定算法  四叉树法

## 核心收获

### 九宫格法和其他AOI算法的对比 

|算法|	核心特点|	最佳适用场景|	慎用场景|
|-----|-----------|---------------|-----------|
|网格法（九宫格）|	简单直接，按位置划分固定格子	| 平坦开阔地图、玩家分布相对均匀（如 MOBA、大逃杀）|	玩家高度聚集（如主城摆摊）、地图地形复杂多障碍|
|十字链表|	基于双向链表维护 X/Y 轴顺序，擅长遍历|	逻辑密集型场景、需要快速 AOI 遍历（如回合制游戏、策略游戏）|	高频创建/删除实体（如频繁进出副本）|
|四叉树|	自适应分区，根据对象密度动态调整	| 地形复杂、对象分布不均匀（如开放世界 RPG、场景中有大量障碍物）|	需要极高性能的实时竞技游戏（因为动态分区和移动更新开销较大）|


###  底向上的 entityCount_ 计数传播
-- 1. 没有采用“递归遍历累加”的常规写法，而是在插入和删除时沿着路径向上更新计数。这种设计让 GetTotalEntityCount() 直接返回 O(1) 的缓存值，避免了每次合并判断时的递归开销。


###  CheckOnSameNodeLeaf 移动剪枝

-- 1. 移动实体时，如果新旧位置仍然落在同一个叶子节点内，就直接更新坐标并返回，省去了“删除 + 重新插入”的完整流程。四叉树的移动开销主要来自删插，而这个剪枝可以不去做重复误用的工作。

### 边界条件与 2 的幂对齐

-- 1.  QuadTreeAOI 的构造函数中主动将世界尺寸向上取整为 2 的幂，并在 DivideToSubChild 中检查 width/height <= 1，防止无限递归。这些细节保证算法在任何输入下都不会崩溃，属于工程健壮性的体现。
